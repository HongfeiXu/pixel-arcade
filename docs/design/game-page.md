# GamePage — 通用游戏容器页面设计文档

## 概述

GamePage 是所有游戏的通用容器页面（路由 `/game/:id`）。它负责游戏的完整生命周期：存档检测 → 倒计时启动 → 游戏进行 → 暂停/恢复 → 游戏结束 → 返回大厅。

## 页面布局

```
┌──────────────────────────────┐
│ [←]              [⏸] [🔇]   │  ← 顶部栏（safe-area-inset-top 内）
├──────────────────────────────┤
│                              │
│         游戏区域              │  ← Canvas，占屏幕高度的 60-65%
│        （Canvas）             │
│                              │
│   [下一个]     ⭐ 12         │  ← 游戏内 HUD（由 Canvas 绘制）
├──────────────────────────────┤
│                              │
│    [←]    [旋转]    [→]      │  ← 虚拟手柄区域（React 组件）
│                              │     占屏幕高度的 35-40%
│         [↓ 软降]             │
│        [⬇ 到底]             │
│                              │
└──────────────────────────────┘
```

### 顶部栏

| 元素 | 位置 | 说明 |
|------|------|------|
| 返回按钮 [←] | 左上角 | 点击后先暂停游戏，再弹出确认弹窗"退出游戏？" |
| 暂停按钮 [⏸] | 右上角偏左 | 切换暂停/继续 |
| 静音按钮 [🔇] | 右上角 | 切换全局 `soundEnabled`，同步到 `pixelarcade_settings` |

顶部栏按钮最小触控区域 44 × 44 pt，使用像素风自绘图标。

## 用户流程

### 1. 进入页面

```
用户从大厅点击游戏卡片
  → 淡入过渡（200ms）进入 GamePage
  → useGame Hook 初始化：
    1. 创建 GameInstance
    2. 调用 init(canvas, config) 绑定 Canvas
    3. 检查 localStorage 是否有存档
```

### 2. 存档恢复（有存档时）

检测到 `pixelarcade_{gameId}_state` 有数据时，显示恢复覆盖层：

```
┌──────────────────────────────┐
│                              │
│      ⭐ 上次获得 8 颗星星     │
│                              │
│     [ 继续游戏 ]              │  ← 主按钮，黄色强调
│     [ 新游戏  ]              │  ← 次按钮，普通样式
│                              │
└──────────────────────────────┘
```

- "继续游戏"：调用 `loadState()` 恢复状态，跳过倒计时，直接进入游戏
- "新游戏"：清除存档，进入倒计时流程

### 3. 倒计时启动

无存档，或选择"新游戏"后：

```
┌──────────────────────────────┐
│                              │
│                              │
│             3                │  ← 48px 像素字体，居中
│                              │     3 → 2 → 1，每个停留 800ms
│                              │     使用 steps() 缩放动画
└──────────────────────────────┘
```

- 倒计时期间 Canvas 显示空棋盘（网格可见），虚拟手柄已渲染但不响应输入
- 倒计时结束后调用 `gameInstance.start()`，首次用户交互（倒计时数字本身的出现）用于解锁 iOS AudioContext
- 倒计时结束的瞬间播放一个短促的"开始"音效

### 4. 游戏进行中

- 虚拟手柄按钮事件通过 `useGame` Hook 转发给 `gameInstance.onInput(action)`
- `onScoreChange` 回调更新 React 侧的星星显示
- `document.visibilitychange`（hidden）触发自动暂停 + 保存状态

### 5. 暂停

点击暂停按钮、点击返回按钮、或切到后台时进入暂停状态。

暂停覆盖层：

```
┌──────────────────────────────┐
│                              │
│          ⏸ 暂停              │
│                              │
│       [ ▶ 继续 ]             │  ← 主按钮
│       [ ↩ 返回大厅 ]         │  ← 次按钮
│                              │
└──────────────────────────────┘
```

- 覆盖层背景：`rgba(26, 26, 46, 0.85)`，参见 [视觉规范](visual-guide.md#覆盖层--弹窗样式)
- "继续"：调用 `gameInstance.resume()`，关闭覆盖层
- "返回大厅"：保存当前状态到 localStorage，调用 `gameInstance.destroy()`，导航回 `/`

### 6. 游戏结束

`onGameOver` 回调触发时：

```
┌──────────────────────────────┐
│                              │
│        游戏结束！             │
│                              │
│     ⭐⭐⭐⭐⭐               │  ← 本局获得的星星，逐个飞入动画
│      共 5 颗星星              │
│                              │
│     🏆 新纪录！               │  ← 仅在刷新最高分时显示
│                              │
│       [ 再来一局 ]            │  ← 主按钮
│       [ 返回大厅 ]            │  ← 次按钮
│                              │
└──────────────────────────────┘
```

- 覆盖层出现前，棋盘先播放"逐行灰化"动画（参见 [俄罗斯方块设计](tetris.md#反馈设计)）
- 灰化完成后覆盖层淡入
- 如果本局分数 > 历史最高分，更新 `pixelarcade_scores` 并显示"新纪录！"
- "再来一局"：清除存档，重新进入倒计时流程
- "返回大厅"：清除存档，导航回 `/`

### 7. 退出确认

点击返回按钮 [←] 时，如果游戏不处于 `idle` 或 `over` 状态：

```
┌──────────────────────────────┐
│                              │
│      确定退出游戏？           │
│     （进度会自动保存）         │
│                              │
│    [ 退出 ]    [ 继续玩 ]    │
│                              │
└──────────────────────────────┘
```

- "退出"：保存状态，destroy，导航回大厅
- "继续玩"：resume，关闭弹窗

## 状态机

```
                    ┌─────────┐
    进入页面 ───────→│  idle   │
                    └────┬────┘
                         │
              ┌──────────┴──────────┐
              │ 有存档？             │
              ├─── 是 → 恢复覆盖层   │
              └─── 否 → 倒计时      │
                         │
                    ┌────▼────┐
   resume ←─────── │ playing │ ──────→ pause
     │              └────┬────┘          │
     │                   │               │
     └───── paused ◄─────┘               │
              │                          │
              └──────────────────────────┘
                         │
                    ┌────▼────┐
                    │  over   │
                    └────┬────┘
                         │
              ┌──────────┴──────────┐
              ├─ 再来一局 → 倒计时   │
              └─ 返回大厅 → 导航 /   │
```

## 与其他模块的关系

| 模块 | 交互方式 |
|------|---------|
| `useGame` Hook | GamePage 使用此 Hook 管理 GameInstance 生命周期 |
| `GamePad` 组件 | GamePage 渲染虚拟手柄，将按钮事件传给 Hook |
| `ScoreBoard` 组件 | GamePage 渲染计分板，数据来自 Hook 的 score 状态 |
| `registry.ts` | GamePage 根据路由 `:id` 从注册表查找并创建对应游戏实例 |
| localStorage | 通过 useGame Hook 间接读写存档和分数 |
