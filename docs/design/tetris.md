# 俄罗斯方块 — 游戏设计文档

## 概述

面向 5 岁幼儿的简化版俄罗斯方块。保留经典玩法核心，降低难度，强化正向反馈。

## 棋盘参数

| 参数 | 值 | 说明 |
|------|----|------|
| 列数 | 10 | |
| 行数 | 18 | |
| 格子尺寸 | 36 pt | 基于 393pt 屏幕宽度计算，左右各留约 16pt 边距 |
| 棋盘总宽度 | 360 pt | |
| 棋盘总高度 | 648 pt | |

## 方块定义

使用 5 种方块，**暂不加入 S 和 Z**（对幼儿来说辨识和放置难度过高）。

### I 型（长条）

```
□□□□
```
颜色：青色 `#00E5FF`

### O 型（方块）

```
□□
□□
```
颜色：黄色 `#FFD600`

### T 型

```
□□□
 □
```
颜色：紫色 `#AA00FF`

### L 型

```
□
□
□□
```
颜色：橙色 `#FF9100`

### J 型

```
 □

□□
```
颜色：蓝色 `#2979FF`

每种方块支持 4 个旋转状态（O 型除外，只有 1 个状态）。旋转使用 **SRS 简化版**，碰壁时尝试 wall kick。

### Wall Kick 规则（简化 SRS）

标准 SRS 每次旋转有 5 个偏移测试位，对幼儿场景简化为 **3 个测试位**：

| 测试顺序 | 偏移 (dx, dy) | 说明 |
|----------|---------------|------|
| 1 | (0, 0) | 原位尝试 |
| 2 | (-1, 0) / (+1, 0) | 向远离墙壁的方向偏移 1 格 |
| 3 | (0, -1) | 向上偏移 1 格 |

- 测试 2 的方向根据方块与墙壁的相对位置自动选择：靠左墙则向右偏移，靠右墙则向左偏移，都不靠则先左后右
- I 型方块因为较长，测试 2 额外允许偏移 2 格
- 3 个测试位全部失败则旋转不生效

## 速度与难度

| 参数 | 值 |
|------|----|
| 初始下落间隔 | 1000 ms（1 秒 1 格） |
| 加速规则 | 暂不实现自适应难度，保持固定速度 |
| 软降（长按下键） | 100 ms/格 |
| Hard Drop | 支持，一键到底 |
| 锁定延迟 | 500 ms（方块触底后有短暂时间可以左右移动和旋转） |

> 后续可扩展：每消 N 行减少下落间隔，最快不低于 300ms。

## 计分规则

采用星星计分制，不使用数字分数，对幼儿更直观。

| 动作 | 奖励 |
|------|------|
| 消除 1 行 | ⭐ × 1 |
| 消除 2 行 | ⭐ × 2 |
| 消除 3 行 | ⭐ × 3 |
| 消除 4 行 | ⭐ × 4 |

> 刻意采用线性计分（而非经典俄罗斯方块的非线性奖励），降低规则理解门槛，让幼儿直观感受"消得多 = 星星多"。

星星在界面上以图标形式展示，同时显示累计数字。

## 操控方案

采用**虚拟按键**方案（方案 1），不依赖手势操作。

### 按钮布局

屏幕下方约 35-40% 区域为操控区：

```
┌──────────────────────────────┐
│                              │
│         游戏区域              │
│        （Canvas）             │
│                              │
│   [下一个]     ⭐ 12         │
├──────────────────────────────┤
│                              │
│    [←]    [旋转]    [→]      │
│                              │
│         [↓ 软降]             │
│        [⬇ 到底]             │
│                              │
└──────────────────────────────┘
```

### 按钮规格

| 参数 | 值 |
|------|----|
| 最小触控区域 | 56 × 56 pt（大于 Apple HIG 建议的 44pt） |
| 按钮样式 | 圆角矩形，像素风描边，按下时有缩放反馈 |
| 长按支持 | 左/右方向键支持长按连续移动，间隔 150ms |

## 游戏启动

进入游戏页面后，显示 **3 → 2 → 1** 倒计时动画，倒计时结束后游戏自动开始。

详见 [GamePage 设计文档](game-page.md)。

## 功能列表（规划清单）

### 核心功能

- [ ] 方块生成（随机包算法，确保均匀分布）
- [ ] 方块移动（左、右、软降）
- [ ] 方块旋转（顺时针，带 wall kick）
- [ ] Hard Drop（一键到底，显示投影）
- [ ] 碰撞检测
- [ ] 行消除（带动画）
- [ ] 下一个方块预览
- [ ] 计分（星星）
- [ ] 游戏结束判定

### 辅助功能

- [ ] 暂停/继续
- [ ] 投影（Ghost Piece）— 半透明显示方块将要落到的位置
- [ ] 游戏结束画面（显示星星总数，重新开始按钮）
- [ ] 状态自动保存（暂停或切后台时存入 localStorage）

## 反馈设计

### 视觉反馈

| 事件 | 效果 |
|------|------|
| 方块落地 | 轻微弹跳动画（50ms） |
| 消行 | 行内格子闪烁 2-3 次后消失，上方行下落 |
| 获得星星 | 星星图标从消除位置飞向计分板，带缩放动画 |
| Hard Drop | 方块瞬间到底，轻微震屏效果 |
| 游戏结束 | 棋盘从底部逐行灰化 |

### 音效

| 事件 | 音效描述 |
|------|---------|
| 移动 | 短促咔哒声 |
| 旋转 | 轻柔转动声 |
| 落地 | 沉闷落地声 |
| 消 1 行 | 清脆消除声 |
| 消 2+ 行 | 更高音调 + 叠加声效 |
| Hard Drop | 重击声 |
| 游戏结束 | 柔和的结束音（不要恐怖或沮丧的声音） |
| 按钮点击 | 轻微反馈音 |

> 全局静音按钮，位于游戏区域右上角。

### 震动

`navigator.vibrate()` — 仅作为增强，iOS 不支持时静默降级。

| 事件 | 震动 |
|------|------|
| 落地 | 10ms |
| 消行 | 20ms |
| Hard Drop | 30ms |

## 随机方块生成

使用 **7-bag 简化版（5-bag）** 算法：将当前 5 种方块放入一个袋子，随机打乱顺序依次取出，取完再放入新一轮。确保不会连续太久不出某种方块。

## 游戏结束条件

新方块生成时，如果生成位置已被占据，则游戏结束。
