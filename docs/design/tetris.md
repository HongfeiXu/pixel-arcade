# 俄罗斯方块 — 游戏设计文档

## 概述

面向 5 岁幼儿的简化版俄罗斯方块。保留经典玩法核心，降低难度，强化正向反馈。

## 棋盘参数

| 参数 | 值 | 说明 |
|------|----|------|
| 列数 | 10 | |
| 行数 | 18 | |
| 格子尺寸 | 36 pt（逻辑值） | 基于 393pt 屏幕宽度计算，左右各留约 16pt 边距。见下方"高度适配"说明 |
| 棋盘总宽度 | 360 pt | |
| 棋盘总高度 | 648 pt | 逻辑高度，实际显示通过 CSS 缩放适配 |

### 高度适配

36pt 格子尺寸是基于屏幕**宽度**推算的最优值，但高度方向无法容纳完整棋盘。以 iPhone 16 Pro（852pt）为例：

```
可用高度 = 852 - 59(safe-top) - 34(safe-bottom) = 759pt
顶栏:    ~60pt
手柄区:  ~200pt
Canvas:  759 - 60 - 200 = ~499pt（可用）
棋盘:    18 × 36 = 648pt（需要）
缺口:    648 - 499 = 149pt
```

**解决策略**：Canvas 按 36pt 格子尺寸渲染（保证像素清晰），然后通过 **CSS `transform: scale()`** 等比缩小至可用空间内。这样宽度方向的边距会变大，但棋盘内容完整可见，且像素风格不受损。

"下一个"预览和星星计分不在 Canvas 内绘制，改由 React 顶栏承载，避免占用棋盘空间。

每种方块支持 4 个旋转状态（O 型不可旋转）。旋转使用 **SRS 简化版**，碰壁时尝试 wall kick。

### Wall Kick 规则（简化 SRS）

标准 SRS 每次旋转有 5 个偏移测试位，对幼儿场景简化为 **3 个测试位**：

| 测试顺序 | 偏移 (dx, dy) | 说明 |
|----------|---------------|------|
| 1 | (0, 0) | 原位尝试 |
| 2 | (-1, 0) / (+1, 0) | 向远离墙壁的方向偏移 1 格 |
| 3 | (0, -1) | 向上偏移 1 格 |

- 测试 2 的方向根据方块与墙壁的相对位置自动选择：靠左墙则向右偏移，靠右墙则向左偏移，都不靠则先左后右
- I 型方块因为较长，测试 2 额外允许偏移 2 格
- 3 个测试位全部失败则旋转不生效

## 方块配色

每个方块颜色需要 3 个层次：主色（填充）、亮色（顶部/左侧高光）、暗色（底部/右侧阴影），模拟像素风格的立体感。

计算规则：亮色 = 主色向白色混合 30%，暗色 = 主色向黑色混合 30%。

| 方块 | 主色 | 亮色（高光） | 暗色（阴影） |
|------|------|-------------|-------------|
| I | `#00E5FF` | `#4CEDFF` | `#00A0B3` |
| O | `#FFD600` | `#FFE24C` | `#B39600` |
| T | `#AA00FF` | `#C44CFF` | `#7700B3` |
| L | `#FF9100` | `#FFB24C` | `#B36600` |
| J | `#2979FF` | `#69A1FF` | `#1D55B3` |

## 速度与难度

| 参数 | 值 |
|------|----|
| 初始下落间隔 | 1000 ms（1 秒 1 格） |
| 加速规则 | 暂不实现自适应难度，保持固定速度 |
| 软降（长按下键） | 100 ms/格 |
| Hard Drop | 支持，一键到底 |
| 锁定延迟 | 500 ms（方块触底后有短暂时间可以左右移动和旋转） |

> 后续可扩展：每消 N 行减少下落间隔，最快不低于 300ms。

## 计分规则

采用星星计分制，不使用数字分数，对幼儿更直观。

| 动作 | 奖励 |
|------|------|
| 消除 1 行 | ⭐ × 1 |
| 消除 2 行 | ⭐ × 2 |
| 消除 3 行 | ⭐ × 3 |
| 消除 4 行 | ⭐ × 4 |

> 刻意采用线性计分（而非经典俄罗斯方块的非线性奖励），降低规则理解门槛，让幼儿直观感受"消得多 = 星星多"。

星星在界面上以图标形式展示，同时显示累计数字。

## 操控方案

采用**虚拟按键**方案（方案 1），不依赖手势操作。

### 按钮布局

屏幕下方为操控区：

```
┌──────────────────────────────┐
│ [←] [下一个]  ⭐ 12   [⏸]   │  ← 顶栏（React）
├──────────────────────────────┤
│                              │
│         游戏区域              │
│      （Canvas 纯棋盘）        │
│                              │
├──────────────────────────────┤
│                              │
│    [←]    [旋转]    [→]      │
│                              │
│         [↓ 软降]             │
│        [⬇ 到底]             │
│                              │
└──────────────────────────────┘
```

### 按钮规格

| 参数 | 值 |
|------|----|
| 最小触控区域 | 56 × 56 pt（大于 Apple HIG 建议的 44pt） |
| 按钮样式 | 圆角矩形，像素风描边，按下时有缩放反馈 |
| 长按支持 | 左/右方向键支持长按连续移动，间隔 150ms |

## 反馈设计

### 视觉反馈

| 事件 | 效果 |
|------|------|
| 方块落地 | 轻微弹跳动画（50ms） |
| 消行 | 行内格子闪烁 2-3 次后消失，上方行下落 |
| 获得星星 | 星星图标从消除位置飞向计分板，带缩放动画 |
| Hard Drop | 方块瞬间到底，轻微震屏效果 |
| 游戏结束 | 棋盘从底部逐行灰化 |

### 音效

| 事件 | 音效描述 |
|------|---------|
| 移动 | 短促咔哒声 |
| 旋转 | 轻柔转动声 |
| 落地 | 沉闷落地声 |
| 消 1 行 | 清脆消除声 |
| 消 2+ 行 | 更高音调 + 叠加声效 |
| Hard Drop | 重击声 |
| 游戏结束 | 柔和的结束音（不要恐怖或沮丧的声音） |

## 随机方块生成

使用 **7-bag 简化版（5-bag）** 算法：将当前 5 种方块放入一个袋子，随机打乱顺序依次取出，取完再放入新一轮。确保不会连续太久不出某种方块。

## 游戏结束条件

新方块生成时，如果生成位置已被占据，则游戏结束。
